Checkpoint 1.2

Provide redundant text links for each active region of a server-side image map.

The "server side" image maps (use ISMAP instead of IMG), do not or can not, habitually, provide any textual information on the coded texts inside the map. If your "server side" image map has links for the A, B, C, D and E sections of your web site, provide an alternative text to the page.

For example, the code “server-side”:

<a href=”img/imgmap1.map”>

 

<img ismap src=”imgmap1.gif” alt=”Please, use the followed links instead of those that are in this image map.” />

 

</a>

 

<br />

 

[ <a href=”a.htm”>Section A</a> |

<a href=”b.htm”>Section B</a> |

<a href=”c.htm”>Section C</a> |

<a href=”d.htm”>Section D</a> |

<a href=”e.htm”>Section E</a> ]

produces the following server-side image map and the correspondent textual equivalent:

[ Section A | Section B | Section C | Section D | Section E ]

Observation: the ALT text that is located in the IMG elemtent tells the user that there is a textual equivalent. It doesn't describes, however, the images by itself. To describe the image map in detail, use the LONGDESC.
