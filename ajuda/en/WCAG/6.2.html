Checkpoint 6.2

Ensure that equivalents for dynamic content are updated when the dynamic content changes.

Your Frame must be based in a HTML document.

Correct way:

<frame name=”frame_photo” src=”home.html”>

where “home.html” must have:

<img src=”home.gif” alt=”Picture of a house.” />

Wrong way:

If you do the frame source the the own image file:

<frame name=”frame_photo” src=”home.gif”>

will be impossible to put textual equivalent to the image.

It would be more apropriated to use the TITLE attribute of the FRAME element to put subtitles or to describe the content of the frame, but the things get difficult when you use the dynamic content that changes every step.

Wrong way:

In this example, the photografy frmae can be created with the followed code:

<frame src=”image-changer.html” title=”picture of a house”>

However, the TITLE attribute equals to a 'picture of a house", is not a apropriated title to be used in a frame once the image changed to a telephone and vice-versa.

Correct way:

A much more apropriated title can change by with the alteration of the image inside the frame.
