Ponto de verificação 6.2

Assegurar que os equivalentes de conteúdo dinâmico sejam atualizados sempre que esse conteúdo mudar.

Construa o Frame sempre baseado num documento HTML.


Nota: Para simular o exemplo de frame usamos uma simples tabela e uma imagem.

A forma acessível de inserir conteúdo num frame é fazer com que a criação do mesmo se baseie num arquivo HTML.

Forma correta

<frame name=”frame_foto” src=”home.html”>

em que “home.html” deve conter o seguinte:

<img src=”home.gif” alt=”Fotografa de uma casa.” />

Forma errada

Se fizer da fonte do frame o próprio arquivo de imagem:

<frame name=”frame_foto” src=”home.gif”>

não vai ser possível dar um equivalente textual à imagem.

Seria mais apropriado usar o atributo “title” do elemento FRAME para legendar ou descrever o conteúdo do frame, mas a dificuldade começa quando se faz uso de conteúdo dinâmico que se modifica a cada passo.

Alterar o conteúdo é possível, mas o mesmo já não se pode dizer do atributo “title” do FRAME e por isso o conteúdo poderá mudar enquanto que o “title” permanece constante. Utilize então, o html fonte de cada nova imagem, essa página, pode conter todos os equivalentes textuais ou informação explicativa necessária à compreensão por parte do usuário.

Forma errada

Neste exemplo, o frame da fotografa pode ser criado com o seguinte código:

<frame src=”image-changer.html” title=”fotografa de uma casa”>

Contudo, o atributo ‘title’ igual à ‘fotografa de uma casa’, não é um título apropriado para o frame uma vez que a imagem se altera de uma casa para um telefone e vice-versa.

Forma correta

Um título mais apropriado pode mudar com a alteração da imagem que se encontra no frame.
