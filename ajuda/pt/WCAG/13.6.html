Ponto de verificação 13.6

Agrupar links relacionados entre si, identificar o grupo (em benefício dos agentes do usuário) e, até que os softwares do usuário se encarreguem de tal função, fornecer um modo de contornar determinado grupo.

Os usuários cegos saltam frequentemente de link em link quando percorrem uma página ou visualizam informação. Quando fazem isto, o “link text” - texto do link (texto que se encontra entre as instruções <A> e </A>) é lido. A barra de navegação é, não raras as vezes, a primeira coisa a ser encontrada numa página. Para usuários de sintetizadores de fala, isto significa ter de ouvir, ler um número grande de links similares em todas as páginas antes de chegar ao conteúdo singular dessa página.

No entanto, quando os links se encontram agrupados em conjuntos lógicos, tais como barras de navegação, eles conseguem manipular uma peça em vez de diversas peças. Desta forma, pode preceder os elementos agrupados por um link que permite ao usuário saltar o conjunto e posicionar-se de forma imediata no início do corpo principal da página. Isto é em tudo semelhante ao que sucede com as pessoas que saltam a leitura dos links quando vêem o mesmo conjunto de informação em cada uma das páginas.

Uma outra forma de agrupar os links é usando o comando MAP. Não é necessário ter uma imagem associada ao comando para poder usá-lo.

<map name=”servicos” alt="Serviços de consultoria do site">

Serviços <br />

<a href=”a.htm”>Serviço A</a> <br />

<a href=”b.htm”>Serviço B</a> <br />

<a href=”c.htm”>Serviço C</a> <br />

<a href=”d.htm”>Serviço D</a> <br />

</map>

Olhar as alternativas ao teclado (ponto de verificação 9.5) e/ou saltar arte em ASCII (ponto de verificação 13.10) para criar alternativas para agrupar e passar por grupos de links.
