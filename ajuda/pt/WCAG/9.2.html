Ponto de verificação 9.2

Assegurar que qualquer elemento dotado de interface própria possa funcionar de modo independente de equipamentos, dispositivos ou qualquer hardware.

Nem todas as pessoas têm computadores que disponham de mouse ou outros dispositivos parecidos. Alguns usuários dependem do teclado, teclados alternativos ou microfone para utilizar o computador.

Links do mapa de imagens:

Forneça equivalentes textuais para áreas de mapas de imagem “client-side”, ou forneça links texto redundantes para mapas de imagens “server-side”.

O acesso via teclado a links e controles de formulário podem ser especificados de algumas maneiras:

 

 

Atalhos de teclado:

Atalhos de teclado permitem aos usuários combinar teclas de atalho para navegar nos links e nos controles de formulário numa página. As teclas de atalho podem ser modificadas consoante os diferentes sistemas operativos, residindo a diferença essencialmente na seqüência de teclas a executar para ativar o atalho. Nas máquinas Windows, as teclas “alt” e “ctrl” são vulgarmente as teclas eleitas; no Macintosh, as teclas “apple” ou “clover leaf”.

Ordem dos Tabs:

A ordem dos "tabs" descreve a (logicamente) ordem de navegação de link para link ou de um campo de formulário para outro campo, usualmente pressionando a tecla “tab”. Teste o seu site navegando somente com a tecla "TAB", para avançar de link em link. Para voltar na ordem dos link use "SHIFT" + "TAB".

Embutir nas interfaces controles independentes dos dispositivos:

Em objetos cujas interfaces não possam ser controladas por nenhuma linguagem de notação. Por exemplo, em HTML: applets, leitores de multimídia ou Flash. Os usuários devem certificar-se que se os objetos importados por si só forneçam interfaces acessíveis ou que pelo menos existe uma alternativa que o faça.
