Ponto de verificação 1.1

Fornecer um equivalente textual a cada elemento não textual (por ex., por meio de “alt” ou “longdesc”, ou como parte do conteúdo do elemento). Isso abrange: imagens, representações gráficas do texto (incluindo símbolos), regiões de mapa de imagem, animações (por ex., GIF animados), applets e objetos programados, arte ASCII, frames, programas interpretáveis, imagens utilizadas como sinalizadores de pontos de enumeração, espaçadores, botões gráficos, sons (reproduzidos ou não com interação do usuário), arquivos de áudio independentes, trilhas áudio de vídeo e trechos de vídeo.

a) Um equivalente textual para imagens e botões gráficos

Exemplo 1

Esta foto é de um carro amarelo e o código pode ser:

<img src=”carro.jpg” alt=”Foto de um carro amarelo.” />

Para uma imagem simples, decorativa ou ocasional, fazer apenas uma breve descrição.


Se a imagem constituir um link, descrever o destino ou o propósito do link – e não a imagem. Neste caso, o desenho da casa é um botão que constitui um link, que liga à página principal. Por isso, o código pode ser o seguinte:

<a href=”carro.htm”>

<img src=”carro.jpg” alt=”Link para a página de entrada.” /> </a>

Exemplo 2

Se a informação contida na imagem é importante para a compreensão da página (ou seja, uma parte do conteúdo informativo pode ser perdida, caso seja retirada a imagem). O atributo LONGDESC foi criado para situações em que a descrição a ser feita deve ser mais longa do que a permitida pelo atributo ALT.

<img src=”montanhas_alasca.jpg” longdesc=”montanhas.htm” alt=”Foto de montanhas no Alasca.” />

Os futuros navegadores ou agentes do usuário irão disponibilizar um link opcional para o arquivo que conterá a descrição, chamado “montanhas.htm”.

Exemplo 3

Se você utiliza o elemento OBJECT para posicionar uma imagem, inclua o texto alternativo no conteúdo do elemento OBJECT, por exemplo:

<object data=”trigo.jpg” type=”image/jpg” height=”100” width=”100”>

Coloque aqui a descrição curta ou longa (ou link para um arquivo contendo uma

descrição longa).

</object>

Observação: O elemento OBJECT ainda não é um formato padrão para imagens. Ele só funciona nos navegadores Mozilla (Mozilla, Firefox e Netscape) e Opera. Observe nos exemplos abaixo como ele se comporta nos navegadores.

b) Um equivalente textual para representações gráficas de texto (incluindo símbolos)

Para imagens que são simplesmente mapas de bits de texto (porque o objetivo é usar tipos de letra especiais ou outro tipo de transformação difícil ou impossível de ser feito com folhas de estilo), forneça o respectivo texto equivalente. Por exemplo:

<img src=”MAG.gif” alt=”MAG Innovision” />

Sempre que existir uma linguagem de notação para símbolos, como é o caso do MathML, use-a em detrimento de um simples equivalente textual. Para mais detalhes, veja o exemplo para o ponto de verificação 3.1.

c) Um equivalente textual para as regiões do mapa de imagens

Para usuários que utilizem navegadores não gráficos ou para alguém que não use um tradicional dispositivo (mouse ou tela touch-screen), uma página que faça uso de um mapa de imagens como o único meio de suporte à navegação poderá ser bastante inacessível. Existe um conjunto de técnicas disponíveis que lhe permitem assegurar que o seu mapa de imagens sejam acessíveis. Todas essas técnicas envolvem o fornecimento de um equivalente textual para as funcionalidades críticas existentes no mapa: nomear as áreas ativas que representam links para outra informação.

e) Um equivalente textual para APPLETS e objetos programáveis

O elemento APPLET está ultrapassado em HTML 4.0. (Em substituição, comece a usar o novo elemento OBJECT). No entanto, ele continua a ser utilizado com freqüência. Se você está modificando um APPLET para torná-lo mais acessível, use o atributo ALT para uma breve descrição da sua ação ou propósito. Caso queira ajudar o público a compreender o propósito do APPLET, inclua uma descrição mais detalhada no conteúdo do elemento APPLET (por exemplo, entre as expressões <APPLET> e </APPLET>).

O código a seguir será produzido independentemente da versão ou da capacidade do seu navegador para mostrá-lo.

<applet code=”Blink.class” width=”500” height=”40” alt=”Applet Java: texto dançante.”>

Se estiver usando um navegador com suporte a JAVA, você poderá ver o texto “Ser ou não

ser, eis a questão.

Ou seria a questão ser ou não ser ...”, flutuando ao longo da tela, em vez deste

parágrafo.

</applet>

g) Um equivalente textual para frames

Use o “title” e se necessário, o atributo “longdesc” no elemento FRAME para descrever mais detalhes sobre o frame ou as suas relações com outros frames. No ponto de verificação 12.2 e 6.5 daremos mais detalhes.

Exemplo

<frame src="principal.html" longdesc="descricao.html" title="Frame com conteúdo principal">

h) Um equivalente textual para scripts

Use o elemento NOSCRIPT para descrever a ação ou substituir a funcionalidade de qualquer script que integre na página.

Por exemplo, se escrever um SCRIPT que produz uma animação apresentando um novo painel sempre que o usuário clique numa dada região da tela ou pressione uma tecla, use o elemento NOSCRIPT para fornecer uma alternativa textual que descreva inteiramente a ação. Por exemplo:

<script type=”text/javascript”>

... script para mostrar e substituir painéis de acordo com a seleção do usuário...

</script>

<noscript>

No primeiro painel, Lucy segura numa bola de futebol. No segundo painel, Charlie Brown

está vendo futebol. E assim sucessivamente...”

</noscript>

i) Um equivalente textual para imagens usadas em substituição dos “bullets” das listas

Aqui está uma lista não ordenada que faz uso de gráficos em substituição das “bullets” de lista criados por efeito pelo elemento <UL>:

 Lápis
 Telefone
 Calendário

O exemplo, em código, apresenta três maneiras diferentes, mas apropriadas, para utilizar o atributo ALT como legenda para os marcadores.

<img src=”blueball.gif” alt=”Item: “ /> Lápis <br /> <img src=”blueball.gif” alt=”* “ /> Telefone <br /> <img src=”blueball.gif” alt=” “ /> Calendário <br />

  1. Para salientar o item ou qualquer outro texto simples, deve ser usado “Item”, especialmente se não existirem muitas entradas na lista.
  2. Para fornecer uma alternativa textual idêntica ao bullet (marcador gráfico) deve ser usado um asterisco “*” ou um hífen “-”.
  3. Para uma lista com grande número de itens, será mais apropriado colocar um espaço em branco entre as aspas do atributo ALT. Isto poupa o usuário de leitor de tela de ouvir, repetidamente, a indicação “Item...”.

j) Um equivalente textual para imagens usadas como spacers (espaçadores)

A criação de um arquivo de imagem muito pequeno e transparente é um truque de layout utilizado por muitos autores. Isto força espaços em branco entre outros objetos, especificando a altura e a largura, ou os atributos hspace e vspace do elemento IMG. O equivalente textual apropriado para um spacer é simplesmente nada, ou seja, um espaço em branco. Por isso:

<img src=”spacer.gif” hspace="100" vspace="10" alt=” “ />

ou

<img src=”spacer.gif” width="5" height="200" alt=” “ />

Observe que existe um espaço em branco entre as marcas de aspas no atributo ALT. Nós preferimos, claro, que para definir seu layout, seja feito uso de notação para folhas de estilo.

k) Um equivalente textual para sons (tocado com ou sem interação do usuário).

Se você produziu um script que causa um aviso sonoro para ser tocado se o visitante da sua página tentar enviar um formulário antes dos campos requeridos estarem preenchidos.

Coloque por favor no seu programa (ou script) a capacidade de escrever uma mensagem na tela que diga algo como:

“Você tentou submeter um formulário incompleto. Preencha, por favor, os campos necessários.”

m) Um equivalente textual para faixas de áudio ou vídeo.

Um equivalente textual para a faixa áudio de um vídeo é similar à transcrição textual de um discurso gravado num arquivo de áudio. Isto pode ser um equivalente textual sincronizado - o qual é o método preferido, ou um simples arquivo de texto, como é aqui exemplificado:

spielflm.de: Os aficionados dos seus filmes gostam de etiquetá-lo como poeta, filósofo e visionário. Saberia o que fazer com a etiqueta?

Werner: Pouco. Eu nunca olhei para mim mesmo. Nunca estive ligado a esse fenômeno pouco sadio, um fenômeno da moda, ser obrigado a analisar-me a mim mesmo. Eu não sei nem sequer a cor dos meus olhos. Até olho para o espelho toda manhã e fico atento, mas só para não me cortar ao fazer a barba. Porém prefiro não me olhar nos olhos. Eu não sei, e Deus é minha testemunha, a cor dos meus olhos.

spielflm.de: Azul-claro, como se constata.

Werner: Tudo bem. Você bem que poderia ter dito “verde”, “castanho” ou “preto”. (ri) De qualquer modo, não acredito em nada do que você diz. Mas: Vou desta vez vou ficar com o boato.
